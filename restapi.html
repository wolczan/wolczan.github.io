.NET assemblies in PowerShell

By loading these assemblies in PowerShell using the Add-Type cmdlet, you make the classes and methods within these assemblies available for use in your PowerShell scripts.
This allows you to automate various SharePoint-related tasks or perform custom SharePoint operations from your PowerShell environment

  • Microsoft.SharePoint.Client.UserProfiles.dll
  • Microsoft.SharePoint.Client.dll
  • Microsoft.SharePoint.Client.Runtime.dll

Load .NET assemblies in PowerShell using the Add-Type cmdlet. These assemblies are related to SharePoint and are used for interacting with SharePoint programmatically

Make sure that the file paths are accurate and that the DLL files exist in the specified locations

Once you've loaded these assemblies, you can use the classes and methods they provide to work with SharePoint data

Microsoft.SharePoint.Client.UserProfiles.dll: This assembly provides classes and methods for working with user profiles in SharePoint. It allows you to retrieve and manage user profile information in a SharePoint environment. You might use this assembly when you need to access and update user profile data for users in SharePoint

Microsoft.SharePoint.Client.dll: This is a core assembly for interacting with SharePoint from a remote client, such as PowerShell or a custom application. It provides classes and methods for performing a wide range of SharePoint operations, including working with lists, libraries, sites, and more. This assembly is often a prerequisite for SharePoint-related tasks in PowerShell

Microsoft.SharePoint.Client.Runtime.dll: This assembly contains runtime components that are essential for the functioning of SharePoint client-side code. It provides support for authentication, communication with SharePoint servers, and other runtime features.
